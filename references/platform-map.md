# 百积木平台能力地图

## 产品分层

- 工作区与身份：用户、工作区、成员、权限、计费和默认运行时。
- 项目与文件：项目、文件读写、Git、导入导出。
- 智能体与技能：配置、会话、模型凭证、技能和自动化。
- 应用运行时：Bundle、模块安装、服务清单、方法调用和网关。
- 托管服务与数据：托管后端、环境、服务令牌、数据库和配置绑定。
- 网站与前端发布：静态站点、预览、正式发布和资源包。
- 自动化编排：工作流、计划、定时任务、事件和渠道消息。
- 本地连接：百积木本地端、Connector、设备授权、Relay、桌面和 shell。
- 平台运营：版本、审核、发布、健康、回滚和审计。

## 判断顺序

1. 判断用户是在使用百积木产品能力，还是只修改本地代码。
2. 确定目标资源所属产品分层。
3. 优先使用本机 `baijimu` CLI 已公开的命令。
4. CLI 未封装时，只使用已确认的 Partner API。
5. 写前读取当前状态，写后回查同一状态源并做必要的端到端验证。

## 常见路由

- 工作区、项目文件和 Git：`workspace`、`project`。
- 智能体会话和消息：`agent`。
- Bundle 发现、发布、安装和升级：`bundle`。
- 模块定义和版本：`module`。
- Runtime 服务发现与调用：`runtime`。
- 托管后端、构建和数据库：`hosted-service`、`rust-build`、`db-profile`。
- 平台应用市场和工作区安装：`platform-app`。
- 本地应用与 Connector：`local-app`，并先确认本机版本实际提供的设备运行面。
- CLI 未覆盖的正式能力：`api`，但必须先确认 Partner API 契约。

## 禁止事项

- 不把源码排障、运维配置或内部发布实现当作普通产品能力。
- 不把网关内存、缓存或旧运行态当作状态源。
- 不在缺少真实 ID、方法定义或权限上下文时调用写接口。
- 不用手工替换生产文件或直接编辑内部配置绕过正式流程。
- 不把第三方技能市场兼容层描述成百积木平台核心能力。
